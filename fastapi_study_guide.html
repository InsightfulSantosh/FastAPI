FastAPI — Complete Study Guide

From zero to production-ready APIs

What is FastAPI?

FastAPI is a modern Python web framework for building APIs. It is built on Starlette (for HTTP) and Pydantic (for data validation), and uses Python type hints throughout to give you editor autocomplete, runtime validation, and auto-generated docs with zero extra work.

Performance
Async-first
On par with Node.js and Go. Powered by async/await and ASGI.
Auto Docs
Swagger + ReDoc
Interactive docs generated automatically from your code at /docs and /redoc.
Validation
Pydantic v2
All request/response data is validated automatically. Errors return clean JSON.
Type Safety
Python hints
Function signatures are the source of truth for routes, params, and bodies.
DI System
Depends()
First-class dependency injection for auth, DB sessions, shared logic.
ASGI Server
Uvicorn
Lightweight, ultra-fast ASGI server. Use Gunicorn + Uvicorn workers in prod.

Routing & Request Types

FastAPI maps Python functions to HTTP methods using decorators. The function signature tells FastAPI where to pull each value from.

HTTP methods

GET

Read / retrieve data. Safe and idempotent. Use for listings and detail views.

POST

Create a resource. Not idempotent — calling twice creates two records.

PUT

Full replace of a resource. Idempotent. Send the complete representation.

PATCH

Partial update. Only the fields you send are changed.

DELETE

Remove a resource. Idempotent.

Path parameters

Declare a path parameter in the URL string and as a function argument with the same name. FastAPI validates the type automatically.

@app.get("/items/{item_id}")
def get_item(item_id: int):
    return {"id": item_id}

# GET /items/42   → {"id": 42}
# GET /items/abc  → 422 Unprocessable Entity (not an int)

Query parameters

Any function parameter that is not in the URL path is treated as a query parameter.

@app.get("/search/")
def search(q: str, page: int = 1, limit: int = 10):
    return {"query": q, "page": page, "limit": limit}

# GET /search/?q=fastapi&page=2&limit=5
# q is required; page and limit have defaults

Request body (POST / PUT / PATCH)

from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float
    in_stock: bool = True

@app.post("/items/", status_code=201)
def create_item(item: Item):
    return item

Mixing path, query, and body

@app.put("/items/{item_id}")
def update_item(item_id: int, item: Item, notify: bool = False):
    # item_id  → path param
    # item     → request body (Pydantic model)
    # notify   → query param with default
    return {"id": item_id, "item": item, "notified": notify}

Response model & status codes

class ItemOut(BaseModel):
    id: int
    name: str

@app.get("/items/{id}", response_model=ItemOut, status_code=200)
def get_item(id: int):
    return ItemOut(id=id, name="Widget")
# Strips any extra fields not in ItemOut before returning

Async routes

import httpx

@app.get("/external/")
async def call_external():
    async with httpx.AsyncClient() as client:
        r = await client.get("https://api.example.com/data")
    return r.json()

Use async def when you have I/O-bound operations (DB, HTTP calls). Use plain def for CPU-bound or synchronous work — FastAPI runs plain functions in a thread pool automatically.

Pydantic & Data Validation

Pydantic models define the shape of your data. FastAPI uses them for request bodies, response models, and settings. If data doesn't match, FastAPI returns a 422 with a detailed error list — no extra code needed.

Defining models

from pydantic import BaseModel, Field, EmailStr
from typing import Optional, List
from datetime import datetime

class User(BaseModel):
    username: str = Field(..., min_length=3, max_length=20, pattern="^[a-z0-9_]+$")
    email: EmailStr
    age: int = Field(..., gt=0, lt=150)
    bio: Optional[str] = None
    tags: List[str] = []
    created_at: datetime = Field(default_factory=datetime.utcnow)

Field constraints — strings

ConstraintExample
min_lengthField(..., min_length=3)
max_lengthField(..., max_length=50)
patternField(..., pattern=r"^\d{5}$")

Field constraints — numbers

ConstraintMeaning
gtGreater than
geGreater than or equal
ltLess than
leLess than or equal

Nested models

class Address(BaseModel):
    street: str
    city: str
    zip_code: str

class Order(BaseModel):
    items: List[str]
    shipping: Address   # nested model

Validators

from pydantic import field_validator, model_validator

class Product(BaseModel):
    name: str
    price: float
    discount: float = 0.0

    @field_validator("name")
    def name_must_not_be_blank(cls, v):
        if not v.strip():
            raise ValueError("Name cannot be blank")
        return v.title()

    @model_validator(mode="after")
    def discount_less_than_price(self):
        if self.discount >= self.price:
            raise ValueError("Discount must be less than price")
        return self

Schema customisation & examples

class Item(BaseModel):
    name: str
    price: float

    model_config = {
        "json_schema_extra": {
            "examples": [{"name": "Laptop", "price": 999.99}]
        }
    }

Pydantic models are also great for environment/settings management: from pydantic_settings import BaseSettings

Dependency Injection

FastAPI's Depends() system lets you declare reusable logic — database sessions, auth checks, config — that is injected into route functions automatically. Dependencies can depend on other dependencies.

Basic dependency

from fastapi import Depends

def get_db():
    db = SessionLocal()
    try:
        yield db          # yield makes it a context manager
    finally:
        db.close()

@app.get("/users/")
def list_users(db = Depends(get_db)):
    return db.query(User).all()

Auth dependency

from fastapi import HTTPException, Security
from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

def get_current_user(token: str = Depends(oauth2_scheme)):
    user = decode_jwt(token)
    if not user:
        raise HTTPException(status_code=401, detail="Invalid token")
    return user

@app.get("/me")
def profile(user = Depends(get_current_user)):
    return user

Chaining dependencies

def get_admin(user = Depends(get_current_user)):
    if user.role != "admin":
        raise HTTPException(403, "Admins only")
    return user

@app.delete("/users/{id}")
def delete_user(id: int, admin = Depends(get_admin)):
    ...

Router-level dependencies

# Apply a dependency to every route in a router
router = APIRouter(
    prefix="/admin",
    dependencies=[Depends(get_admin)]
)

Use yield in a dependency to get setup/teardown behaviour (like a context manager). Code before yield runs before the request; code after runs after the response is sent.

Caching — use_cache

# By default, the same dependency called multiple times in one request
# is only executed ONCE (result is cached per request).
# To disable this:
Depends(get_db, use_cache=False)

Error Handling

FastAPI provides several layers of error handling. Use HTTPException for expected HTTP errors, and exception handlers for unexpected ones or to customise the response format.

HTTPException — the standard approach

from fastapi import HTTPException

@app.get("/items/{id}")
def get_item(id: int):
    item = db.get(id)
    if not item:
        raise HTTPException(
            status_code=404,
            detail="Item not found",
            headers={"X-Error": "item-missing"}
        )
    return item

Custom exception class + handler

from fastapi import Request
from fastapi.responses import JSONResponse

class InsufficientFundsError(Exception):
    def __init__(self, amount: float):
        self.amount = amount

@app.exception_handler(InsufficientFundsError)
async def handle_insufficient_funds(request: Request, exc: InsufficientFundsError):
    return JSONResponse(
        status_code=402,
        content={"error": "insufficient_funds", "needed": exc.amount}
    )

Override default validation error handler

from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_exception_handler(request: Request, exc: RequestValidationError):
    return JSONResponse(
        status_code=422,
        content={"errors": exc.errors(), "body": exc.body}
    )

Advanced Topics

Middleware

Middleware wraps every request and response. Use it for logging, timing, adding headers, or rate limiting.

from starlette.middleware.base import BaseHTTPMiddleware
import time

class TimingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        start = time.time()
        response = await call_next(request)
        response.headers["X-Process-Time"] = str(time.time() - start)
        return response

app.add_middleware(TimingMiddleware)

CORS

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://myapp.com"],  # or ["*"] for dev only
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

Background tasks

Run slow tasks (email, webhooks, heavy computation) after the response is returned, without making the client wait.

from fastapi import BackgroundTasks

def send_welcome_email(email: str):
    # runs after response is sent
    mailer.send(email, subject="Welcome!")

@app.post("/register/")
def register(email: str, bg: BackgroundTasks):
    bg.add_task(send_welcome_email, email)
    return {"status": "registered"}

WebSockets

from fastapi import WebSocket, WebSocketDisconnect

@app.websocket("/ws/{client_id}")
async def ws_endpoint(ws: WebSocket, client_id: str):
    await ws.accept()
    try:
        while True:
            data = await ws.receive_text()
            await ws.send_text(f"Echo [{client_id}]: {data}")
    except WebSocketDisconnect:
        print(f"Client {client_id} disconnected")

File uploads

from fastapi import UploadFile, File

@app.post("/upload/")
async def upload(file: UploadFile = File(...)):
    content = await file.read()
    return {"filename": file.filename, "size": len(content)}

Database with SQLAlchemy

from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.orm import DeclarativeBase, Session

DATABASE_URL = "sqlite:///./app.db"
engine = create_engine(DATABASE_URL)

class Base(DeclarativeBase): pass

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True, index=True)
    name = Column(String, index=True)

Base.metadata.create_all(bind=engine)

def get_db():
    with Session(engine) as db:
        yield db

Anatomy of a FastAPI route

@app.post(                       # HTTP method
    "/items/{category}",            # path (with path param)
    response_model=ItemOut,         # output schema
    status_code=201,               # success status
    tags=["items"],               # Swagger grouping
    summary="Create an item",      # Swagger title
    dependencies=[Depends(auth)],   # router-level deps
)
async def create_item(
    category: str,                  # path param
    q: Optional[str] = None,       # query param
    item: ItemIn,                   # request body
    db = Depends(get_db),           # injected dependency
) -> ItemOut:
    ...
